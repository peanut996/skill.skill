# skill.skill

一个人物/视角类 skill 的导航仓库。

不是把所有人硬塞进一个人格大杂烩，
而是把不同的 persona-distill skill 作为独立作品来收录、分类、索引和发现。

</div>

---

## 这是什么

skill.skill 现在先做成一个“合集 / 导航仓库”。

它的目标不是替代每个独立 skill 仓库，而是：

- 收录值得参考的人物 / 视角类 skill
- 用统一结构帮你快速理解每个 skill 在做什么
- 给出安装方式、适用场景、风格标签和仓库链接
- 后续再逐步补充投稿规范、分类标准和模板

你可以把它理解成：

- 一个中文语境下的 persona-distill skill 索引
- 一个方便发现优秀 skill 的入口
- 一个后续可扩展成收藏馆 / curated list 的基础仓库

---

## 收录原则（第一版）

当前优先收录这类项目：

- 以某个人物、创作者、公众人物或方法论视角为核心
- 不只是语录摘抄，而是尽量抽出“认知框架 / 表达 DNA / 决策模式”
- 有清晰 README，能让人快速理解这个 skill 的用途
- 最好有公开仓库链接，便于访问与安装

当前不追求“大而全”，先追求：

- 结构清楚
- 风格统一
- 条目可读
- 后续方便继续扩展

## 为什么先做导航仓库

因为这件事如果一开始就把别人的 skill 全部拷进来，问题会很多：

- 维护成本高
- 原项目更新后不同步
- 容易变成资料堆而不是好用索引
- 仓库边界不清楚

先做导航仓库的好处是：

- 上线快
- 结构清楚
- 方便持续加条目
- 后面要不要做模板、投稿、专题分类，都很好接

所以第一阶段先做：

1. README 导航页
2. 分类与收录说明
3. 已收录项目列表
4. 后续投稿 / 扩展空间

## 说明

本仓库目前是“索引 / 导航”性质，默认尊重原作者仓库的独立性。

也就是说：

- 这里优先放链接与介绍
- 原始内容仍以原仓库为准
- 后续如果要做更深层的聚合，再单独讨论规则

---

如果说独立 skill 仓库像一本本小册子，
那 skill.skill 现在先做的，是书架，不是把所有书拆了重新装订。
