## 这是什么

skill.skill 现在先做成一个“人物 / 视角类 skill 的中文索引仓库”。

它的目标不是替代每个独立 skill 仓库，而是：

- 收录值得参考的人物 / 视角 / 方法论类 skill
- 用统一格式帮助快速理解每个 skill 在做什么
- 给出仓库链接、适用场景、关键词与简短说明
- 作为导航页把用户导向原作者仓库
- 后续逐步补充投稿规范、分类体系和模板

你可以把它理解成：

- 一个 persona-distill skill 的中文入口
- 一个方便发现优秀 skill 的导航页
- 一个后续可扩展成收藏馆 / awesome list 的基础仓库

---

## 收录标准

当前优先收录这类项目：

- 以某个人物、创作者、公众人物或方法论视角为核心
- 不只是语录摘抄，而是尽量抽出认知框架、表达 DNA 或决策模式
- README 结构清晰，能快速说明 skill 的用途
- 有公开仓库链接，便于访问、验证和后续维护

当前第一阶段不追求大而全，先追求：

- 结构清楚
- 条目可读
- 便于扩展
- 方便持续维护

## 为什么先做导航仓库

如果一开始就把别人的 skill 原始内容搬进来，会很容易遇到这些问题：

- 维护成本高
- 原项目更新后不同步
- 容易变成资料堆，而不是好用索引
- 仓库边界不清楚
- 后续很容易搞混“索引仓库”和“内容仓库”的职责

先做导航仓库的好处是：

- 上线快
- 结构清楚
- 方便持续加条目
- 后面要做模板、投稿、专题分类都很好接

所以第一阶段先做：

1. README 导航页
2. 分类与收录说明
3. 已收录项目列表
4. 后续投稿 / 扩展空间

- 这里优先放链接、介绍、标签和分类
